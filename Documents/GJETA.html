📋 Problem Statement

Why Automate Attendance?

Research shows attendance is the single strongest predictor of college grades. Traditional manual methods are slow, prone to impersonation, and produce insecure, easily-modified records.

😴 Issue 1 — Manual Roll Calls

Takes 5–10 minutes per lecture. Students can answer for absent peers. Wasteful of valuable lecture time.

Risk: Inaccurate records & time waste
🎭 Issue 2 — Impersonation

Students can sign in for absent classmates. Biometric systems like fingerprints require physical access and can be spoofed.

Risk: False attendance records
📝 Issue 3 — Insecure Records

Excel-based records can be modified by anyone with access. No audit trail, no authentication mechanism.

Risk: Unauthorized data modification
📸 Issue 4 — Limited Camera Coverage

Single fixed cameras create blind spots in large classrooms. Students at edges or corners may be missed.

Risk: Incomplete attendance capture
🎯 Prior Work Limitations & FaceAttend Solutions
❌ LBPH-based systems: Fail with illumination/pose changes. Only frontal images supported.
✅ FaceAttend: DeepFace handles illumination & angle variations. Side-view training data included.
❌ Single camera: Blind spots in large rooms. Coverage limited.
✅ FaceAttend: Sliding rail camera architecture covers full room irrespective of class size.
❌ Excel storage: Editable by unauthorized users. No access control.
✅ FaceAttend: MySQL + CodeIgniter API with 3-factor auth (username + password + secret key).
❌ Small datasets: Poor recognition for under-represented students.
✅ FaceAttend: Image augmentation (10× expansion) with Gaussian blur, noise, rotation.

🧠 Multi-Task CNN · Detection Phase

MTCNN Face Detection

The Multitask Cascaded Convolutional Neural Network uses 3 progressively refined networks to detect faces, localize them, and identify 5 key facial landmarks — enabling precise alignment before embedding.

📐 MTCNN Pipeline: Input image → Resize pyramid → P-Net (proposals) → R-Net (refinement) → O-Net (output + landmarks)
🏗️ Three-Stage Architecture
Stage 1 — P-Net (Proposal Network)
Input: 12×12. Generates candidate face windows using a fully convolutional network. Applies NMS (Non-Maximum Suppression) to remove overlapping candidates.
Stage 2 — R-Net (Refine Network)
Input: 24×24. Rejects false candidates from P-Net. Bounding box regression calibrates window positions. Further NMS applied.
Stage 3 — O-Net (Output Network)
Input: 48×48. Outputs 5 facial landmarks (eyes, nose, mouth corners). Final face classification + bounding box. Highest precision stage.

🔬 Feature Extraction · DeepFace Framework

FaceNet-512 Embeddings

FaceNet uses triplet loss to learn a 512-dimensional embedding where same-identity faces cluster together and different-identity faces are pushed apart. This enables robust recognition across lighting, angle, and expression variations.

🧮 Triplet Loss Concept
Triplet: (Anchor, Positive, Negative)
Anchor = reference face · Positive = same person · Negative = different person
‖f(anchor) − f(positive)‖² + α < ‖f(anchor) − f(negative)‖²

The loss minimizes distance between anchor & positive (same person), while maximizing distance between anchor & negative (different person). Margin α prevents trivial solutions.

📊 Model Comparison (DeepFace Framework)
🔄 How DeepFace Handles Recognition
  • Framework wraps multiple SOTA models: VGG-Face, FaceNet, OpenFace, DeepFace, DeepID2, Dlib
  • FaceAttend uses FaceNet-512 — best accuracy among options, 512-D output vector
  • Deepface extracts & stores embeddings beforehand as representations_facenet512.pkl
  • Find method: compares detected face against all stored embeddings in database
  • Similarity metrics: cosine similarity, Euclidean distance, Euclidean L2
  • Closest match → student registration number extracted from file path → attendance marked
🖼️ Dataset Enhancement · Imgaug

Image Augmentation

Only 3 photos per student were captured. CNNs need more data to generalize well. The Imgaug library applies random transformations to expand each student's data 10×, dramatically improving recognition accuracy.

📈 Impact: Before augmentation — 1 student missed. After augmentation — ALL 30 students correctly identified and marked.

📐 Similarity Metric · Cosine Similarity

Cosine Similarity Calculator

FaceAttend uses cosine similarity to compare 512-dimensional face embedding vectors. Two vectors pointing in the same direction (same person) produce a score near 1.0; different people produce scores near 0 or below.

📐 Formula: cos(θ) = (A · B) / (‖A‖ × ‖B‖) where A, B are 512-D face embedding vectors

🔗 Security & Access · CodeIgniter

Secure API & Database

A PHP CodeIgniter-based REST API controls access to all attendance records. Three-factor authentication ensures only authorized users — lecturers, management, parents — can view or export data.

🔐 Three-Factor Authentication
Factor 1 — Username
Chosen by the user during registration.
Factor 2 — Password
Chosen by the user, stored securely in DB.
Factor 3 — Secret Key
Auto-generated by API, stored in database. Cannot be guessed.
📡 API Methods
GET /attendance?class_id={id}
Returns JSON attendance report for class
POST /authenticate
Validates 3-factor credentials, creates session

🔍 Limitations & Future Work

Research Gaps

Even a high-performing system has areas for improvement. The paper honestly identifies current limitations and leaves opportunities for future research.

01
👥 Small Sample Size
Only 30 students were tested. Real university classes may have 100–500+ students. Scalability and accuracy at scale need further validation with larger datasets.
02
⚡ Real-Time Processing Speed
Processing batch images (multiple class photos) through MTCNN + FaceNet-512 is computationally intensive. Real-time processing for large classes would require GPU acceleration or edge computing.
03
🕶️ Occlusion & Mask Handling
Post-pandemic, students wearing face masks significantly reduces recognition accuracy. The system was not tested under such conditions. Mask-aware models would be needed.
04
🔒 Privacy & GDPR Compliance
Storing facial embeddings and attendance data raises privacy concerns. Compliance with data protection laws (NDPR in Nigeria, GDPR in EU) is not addressed. Consent frameworks needed.
05
🌡️ Lighting & Environmental Variations
System tested in controlled lab settings. Outdoor venues, poor lighting, or projector glare in lecture rooms could degrade MTCNN detection performance significantly.
06
🤖 Deepfake Spoofing Vulnerability
Students could potentially spoof the system using printed photos or digital displays. Liveness detection (blink detection, depth sensing) would be a critical addition for production deployment.
